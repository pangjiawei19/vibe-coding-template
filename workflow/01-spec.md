# 背景知识
为了帮助你更好的生成 spec，我已为你提供：
1. 项目代码（如果有）
2. 概要设计文档：`@summary.md`（如果有）
3. 也可以直接列出需求点，例如：
    1. 我们在 backend 中增加一个量化交易模块，先实现整个功能的后端部分，不用关心前端。因为只做后端，所以使用的时候是不是可以通过直接运行 python xxx 去启动相关的功能
    2. 该系统支持多个量化交易策略，目前包括：趋势策略、均线策略、布林带策略、均仓策略
    3. 整个系统的核心功能有：回测功能（基于历史数据验证策略效果）、投资决策（生成指定日期的投资建议）
    4. 投资和回测等都是以天为单位，即每个标的每日的收盘价。不用考虑一天内的变化，但未来可能要迭代到支持更细粒度的时间数据
    5. 要有业绩评估指标体系，包括年化收益率、年化波动率、夏普比率、最大回撤、卡玛比率等
    6. 数据源使用 akshare，获取的数据最好能存储到数据库中，这样不需要每次执行时都去获取
    7. 关于 akshare 的使用，要使用最准确地 api。我们的分析针对的标的有股票、指数、etf 等，好像 akshare 有不同的 api，像 stock_zh_index_daily 是不是就可以直接用
    8. 有几个问题需要考虑解决方案：
        a. 指数的值往往都很大，计算时要考虑超限的问题
        b. 对于股票之类的标的，好像有因为分红导致净值下跌，这在做回测的时候就有问题了，看看怎么办
4. 项目理解文档：`@docs/ `目录的文档
